## Background

When I joined the company as a UI Designer in 2022, the product already had a design system created by a third-party team. However, as the product continued to grow, the system was not regularly maintained and lacked proper documentation.

This made it harder for designers and product owners to use components consistently across different product areas.

## Problem Statement

The design system existed, but it was not functioning as a reliable source of truth. Similar components were often recreated differently across multiple Figma files, while new updates were not always reflected in the documentation.

Without clear ownership and component guidelines, designers had to make repeated decisions from scratch, which slowed down the workflow and created inconsistencies across the product.

### Key Issue
- No dedicated design system management
- Lack of component ownership and control
- Lack of updated documentation
- Inconsistent component usage across files

## Goal & Scope

To create a more structured, accessible, and maintainable design system documentation that helps designers and product owners use UI components more consistently.

## Process & Solution

### Step 1 / Restructure Documentation
I started by breaking down combined component modules into separate sections, making each component easier to find, update, and reuse.

  src="/images/projects/design-system/ds_img1.svg"
/>

### Step 2  / Recreate & Standardize Components
I redesigned several components to simplify usage and improve consistency across product design files, including states, colors, and component variations.

<ProjectImage
  src="/images/projects/design-system/ds_img2.svg"
/>

### Step 3  / Audit & Document Component Usage
Audited components used in ongoing projects and documented their anatomy, selected states, and usage references to make future updates easier to maintain.

<ProjectImage
  src="/images/projects/design-system/ds_img3.svg"
/>

## Summary

After restructuring and re-documenting the components, the design system became easier to maintain, reference, and reuse. Designers could work with clearer component standards, while product owners had a more accessible reference when reviewing UI decisions.

Although the process is still ongoing, this improvement helped build a stronger foundation for UI consistency, design collaboration, and future product scalability
